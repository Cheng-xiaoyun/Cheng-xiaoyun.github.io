Attention机制

categories:neural-network basic

Attention机制其实就是一系列注意力分配系数，也就是一系列权重参数罢了。

注意力模型就是要从序列中学习到每一个元素的重要程度，然后按重要程度将元素合并。

换一个角度来理解，我们将attention机制看做软寻址。就是说序列中每一个元素都由key(地址)和value(元素)数据对存储在存储器里，当有query=key的查询时，需要取出元素的value值(也即query查询的attention值)，与传统的寻址不一样，它不是按照地址取出值的，它是通过计算key与query的相似度来完成寻址。这就是所谓的软寻址，它可能会把所有地址(key)的值(value)取出来，上步计算出的相似度决定了取出来值的重要程度，然后按重要程度合并value值得到attention值，此处的合并指的是加权求和。

优点:
一步到位的全局联系捕捉

上文说了一些，attention机制可以灵活的捕捉全局和局部的联系，而且是一步到位的。另一方面从attention函数就可以看出来，它先是进行序列的每一个元素与其他元素的对比，在这个过程中每一个元素间的距离都是一，因此它比时间序列RNNs的一步步递推得到长期依赖关系好的多，越长的序列RNNs捕捉长期依赖关系就越弱。
并行计算减少模型训练时间

Attention机制每一步计算不依赖于上一步的计算结果，因此可以和CNN一样并行处理。但是CNN也只是每次捕捉局部信息，通过层叠来获取全局的联系增强视野。

缺点:
缺点很明显，attention机制不是一个”distance-aware”的，它不能捕捉语序顺序(这里是语序哦，就是元素的顺序)。这在NLP中是比较糟糕的，自然语言的语序是包含太多的信息。如果确实了这方面的信息，结果往往会是打折扣的。说到底，attention机制就是一个精致的”词袋”模型。所以有时候我就在想，在NLP任务中，我把分词向量化后计算一波TF-IDF是不是会和这种attention机制取得一样的效果呢? 当然这个缺点也好搞定，我在添加位置信息就好了。所以就有了 position-embedding(位置向量)的概念了，这里就不细说了。

Attention机制说大了就一句话，分配权重系数。

RNN的梯度下降&使用GRU
categories:neural-network basic

长期的依赖关系：前面的单词，对句子后面的单词有影响。
RNN不擅长捕获这种长期的关系，
层数很深的时候，反向传播，后面层的输出误差，很难影响前面层的权重。
梯度爆炸很容易发现，你的参数会随着指数增加。（解决办法：梯度修剪，观察你的梯度向量，如果他高于某个数值，就缩放梯度向量）
但是梯度消失很难解决。

GRU：门控循环控制单元
能够解决梯度消失问题，捕获长期依赖关系。

神经网络基本结构&语言模型的构建

categories:neural-network basic

one to one：不重要，没什么可说的
one to many：比如音乐生成（序列模型）
many to one：比如情感分析，输入一段评价，生成一个星级的负面或正面。
many to many（输入和输出长度一致）
many to many（输入和输出长度不一致）：比如机器翻译，一段文字翻译为另外一种语言，encoder-decoder。
注意力机制：
把他们组合在一起，就可以生成各种各样的模型。

语言模型：输入一个文本序列，语言模型用y表示x，并且估计每个句子中，各个单词出现的可能性。
如何构建语言模型：
第一个时间步：
输入o向量。然后利用softmax预测第一个词的概率。而不去管结果。softmax可能会输出10000（你的词典大小）种可能。
第二个的时间步：
传入正确的第一个单词和上一步预测的单词，计算出序列的下一个单词是什么，通过之前的输入，计算出字典中的每一个词的概率。
RNN：考虑前面得到的所有词，给出下一个词的分布，从左到右的预测每个词。

训练一个好的模型：
定义一个代价函数，也就是损失函数。整体损失就是每一步损失相加起来。

使用hexo搭建博客并上传到github

Posted on 2019-09-13 | In skilledness

丑小鸭今天第一次使用hexo搭建博客，记录一下。

前提：已安装git、注册github账号。

  1. 首先要安装node.js，最好安装最新版本。
    博主第一次安装的v4版本，结果后面在使用hexo命令时报一下错误：
    SyntaxError: Unexpected token =at exports.runInThisContext (vm.js:53:16)
    然后根据网友提示升级版本：

    但是并没有成功。
    于是乎卸载重新安装最新版本。

    确认nodejs安装成功就直接在cmd里node –version或在git bash里面输入node回车（命令行变成一个右箭头就是成功。ctrl+c退出）

  2. 安装hexo。
    （1） 利用 npm 命令即可安装。（在任意位置点击鼠标右键，选择 Git bash ）

    npm install -g hexo-cli 或者 npm install -g hexo

    （2）创建hexo文件夹：
    随便在你指定的文件夹下（如 D:\hexo ），(在 D:\hexo内点击鼠标右键，选择 Git bash )，Hexo 即会自动在目标文件夹建立网站所需要的所有文件。
    （3）hexo init
    （4）npm install
    （5） 启动本地服务。现在已经搭建起本地的hexo博客了，执行以下命令(在 D:\hexo )。①hexo generate（或者hexo g）②hexo server（或者 hero s）
    然后到浏览器输入 localhost:4000 看看（如果无法访问，请查看你的4000端口是否被占用）。
    至此，本地博客已经搭建起来了，只是本地的，接下来关联GitHub。

  3. 在github中新建一个public的仓库。
    名字命名为yourname.github.io。（你的github名字加.github.io）

  4. hexo关联GitHub。
    找到你存放Hexo的文件夹（如 D:\hexo ），在里面找到_config.yml文件，用notepad++打开，直接拖到最后，可以看到。

    改成以下，特别注意，在：的后面是有一个空格的，千万要小心，不然后出错的，其中yourname即在github上面的用户名

    保存后，然后在当前文件夹（ D:\hexo ）打开Git Bash Here
    $ hexo g
    $ hexo d
    如果出现以下异常
    ERROR Deployer not found: git
    尝试输入以下命令：
    $ npm install hexo-deployer-git –save
    再重新执行刚刚的两条命令
    这时候如果弹出一个对话框，输入在guthub上面的用户名和密码即可
    这时候我们就可以在浏览器输入http://yourname.github.io（yourname替换成github上的名称）就可以看到博客已经搭建成功了。

  5. 更改hexo样式
    ① 首先在浏览器https://hexo.io/themes/进入github官方主题界面。里面有非常多的主题，选择一个自己喜欢的，点进去 ，就比如说我选了这个：cyanstyle ,点击去之后就会进入这个主题拥有者的博客。然后找到博主的仓库，进入他的仓库，在他的仓库中搜索这个样式的名称，找到对应的下载地址，我找的是（https://github.com/wizardforcel/hexo-theme-cyanstyle.git）
    ②进入你的Hexo地址，比如我的是（ D:\hexo ）打开Git Bash Here，输入
    git clone https://github.com/wizardforcel/hexo-theme-cyanstyle.git
    然后将下载下来的这个主题放入你的D:\Hexo\themes文件夹中，给他命名（如cyanstyle）
    每个主题样式的目录如下：

    ③更改一下hexo目录（ D:\hexo ）下的配置文件_config.yml，主题名修改一下即可

    ④然后清空一下public缓存：hexo clean
    ⑤然后重新生成datebase：hexo g
    ⑥ hexo s 浏览器打开本地仓库（localhost:4000你会发现本地仓库已经变为自己刚刚更改的主题了 ）
    ⑦然后上传到远程仓库就行了：hexo d 在浏览器输入域名http://yourname.github.io就可以看到已经更改的主题。

  6. 上传你的第一篇博客。
    你可以使用CSDN网上面提供的MarkDown编辑器 http://write.blog.csdn.net/mdeditor (在csdn博客写新文章的时候能切换到这种编辑器)，写好文章后，找到菜单栏的“导出到本地”选项 。 以md格式导出到本地，然后copy该md文件，粘贴到你当初建的博客站点（ D:\hexo ）文件夹下的source\ _posts目录下，一个md文件对应一篇博客文章。

接着在你的文件夹（ D:\hexo ）下右键空白处，选择Git Bash Here，
输入：hexo generate（hexo g），回车，生成静态页面，
再输入：hexo server（hexo s），回车，到localhost:4000预览博客效果，
最后输入：hexo deploy（hexo d），回车，同步到github上去就行了。
